Why TDD

왜 지금 TDD인가

테스트 없이 개발하면 나중에 더 많은 시간을 쏟게 됩니다. TDD는 처음부터 품질을 코드에 내재화하는 방법입니다.

40%
TDD 적용 시 버그 감소율
TDD를 적용한 프로젝트는 그렇지 않은 프로젝트 대비 평균 40% 이상 버그가 줄어들고, 유지보수 비용도 크게 감소합니다.
IEEE Software · TDD Impact Study 2023
2배
리팩토링 속도 향상
테스트 코드가 있으면 코드 구조를 바꿀 때 두려움이 사라집니다. 테스트가 안전망이 되어 리팩토링 속도가 2배 이상 빨라집니다.
Martin Fowler · Refactoring 2nd Edition
3일
실무 프로젝트 적용까지 걸리는 시간
단위 테스트 작성, Mock 객체 활용, 테스트 자동화까지 — 3일 과정으로 실제 프로젝트에 바로 적용 가능한 TDD 역량을 확보합니다.

Curriculum

3일 커리큘럼

TDD 개념 이해부터 Mock 객체 활용, 테스트 자동화까지 — 매일 실습 산출물을 만들며 체득합니다.

Day 1 · 테스트와 TDD 이해 7H

"TDD가 무엇이고 왜 써야 하는지" 이해하는 날

  • 소프트웨어 테스트 개념
    테스트의 목적과 종류, 소프트웨어 품질과 테스트의 관계 이해
  • 단위 테스트 개념 · 테스트 프레임워크 이해
    단위 테스트의 정의와 작성 원칙, JUnit / PyTest 프레임워크 사용법
  • TDD 개발 사이클 (Red-Green-Refactor)
    실패하는 테스트 작성 → 코드 구현 → 리팩토링의 TDD 3단계 사이클 이해 및 실습
  • 테스트 코드 작성 원칙
    좋은 테스트 코드의 조건(FIRST 원칙)과 테스트 가독성 향상 방법
    테스트 코드 작성 단위 테스트 실습
Day 2 · TDD 기반 개발 7H

"TDD로 실제 기능을 개발하는" 날

  • TDD 개발 프로세스
    요구사항 분석부터 테스트 설계, 구현, 리팩토링까지 TDD 전체 개발 흐름 실습
  • 테스트 설계 방법 · Mock 객체 활용
    경계값 분석, 동등 분할 등 테스트 설계 기법과 Mock 객체를 활용한 의존성 분리 실습
  • 테스트 코드 구조 설계 · 테스트 커버리지 이해
    유지보수 가능한 테스트 코드 구조 설계 방법과 커버리지 측정 도구 활용
    TDD 기반 기능 구현 Mock 활용 테스트 실습
Day 3 · 테스트 기반 코드 개선 7H

"테스트로 코드를 개선하고 자동화하는" 날

  • 테스트 가능한 코드 설계
    의존성 주입, 단일 책임 원칙 등 테스트하기 쉬운 코드 구조 설계 방법
  • 테스트 구조 개선
    중복 제거, 테스트 픽스처 활용 등 기존 테스트 코드의 구조를 개선하는 방법
  • 테스트 자동화 개념 · 테스트 유지 관리 방법
    CI/CD 파이프라인과 테스트 자동화 연계, 장기적인 테스트 유지 관리 전략
    테스트 기반 기능 구현 테스트 코드 개선 테스트 커버리지 분석

Why This Course

다른 개발 교육과 무엇이 다릅니까?

TDD 개념 강의로 끝나지 않습니다. Red-Green-Refactor를 3일 내내 반복하며 몸으로 익히는 과정입니다.

🎯

3일 내내 반복하는 TDD 사이클

Red-Green-Refactor 사이클을 매 세션마다 반복 실습합니다. 개념을 듣는 것과 직접 반복하는 것은 완전히 다른 수준의 학습 효과를 만들어냅니다.

🔍

실습비중 60%

이론 40%, 실습 60%로 구성됩니다. 테스트 코드 작성, Mock 객체 활용, 테스트 커버리지 분석까지 강의실에서 직접 코드를 만들고 개선합니다.

⚡

Day별 완성 산출물

단위 테스트 코드, TDD 기반 기능 구현 결과물, 테스트 커버리지 분석 결과까지 — 매일 손으로 만들어 과정 종료 후 바로 프로젝트에 적용할 수 있습니다.

🧭

테스트 자동화까지 연결

단순 테스트 작성에서 끝나지 않습니다. CI/CD 파이프라인과의 연계, 테스트 유지 관리 전략까지 다루어 지속 가능한 TDD 문화를 만들 수 있습니다.
